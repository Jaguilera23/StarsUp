CREADORES

Como ingeniero mecánico, nunca había considerado la posibilidad redirigir mis conocimientos hacia la transformación digital. Con la necesidad de salir de mi zona de confort decidí probar suerte con pequeños cursos teniendo cada vez mas curiosidad por el desarrollo Front-End. Fui descubriendo un mundo de posibilidades, tablas, cards, pages, frameworks, integraciones y otras tantas cosas que se que le encontrare un uso funcional.
La idea para mi proyecto de cursada Front-End era simular una página web relacionada a un cryptoexchange donde me pudiera permitir exigirme al máximo por la cantidad de recursos visuales que tienen este tipo de sitios. Mi intención desde siempre fue tener un proyecto escalable para ir aplicando distintos stacks tecnológicos en los que vaya adquiriendo conocimientos y poder ampliar este recurso como proyecto.
Durante el desarrollo, me dediqué a crear una variedad de elementos, componentes, iconos, pages, dedicándole muchas horas de código, permitiéndome practicar, equivocarme y aprender.
Con Luis pudimos trabajar en equipo bajo una metodología ágil, planificar y ejecutar cada etapa del proyecto de manera eficiente, simulamos plazos establecidos para alcanzar los objetivos que nos habíamos propuesto y divertirnos mientras aplicábamos nuestros conocimientos y tener una retroalimentación bilateral.
Esta experiencia no solo me permitió crecer profesionalmente aumentar skills y algo a lo que quiero dedicarme de ahora en adelante. Aprendí que siempre hay espacio para crecer y expandirse en nuevos territorios.
Se que este es solo el inicio de un camino donde la retroalimentación es constante y todos los dias son un reto.
Jose Aguilera
Front Developer - Scrum

Siempre busco diseñar proyectos que tengan un gran impacto visual, pero sin dejar de lado la importancia de la factibilidad y accesibilidad.
A lo largo de mi carrera, he tenido la oportunidad de trabajar con equipos de desarrolladores seniors y experimentados.
Pero recientemente, un amigo que llevaba unos tres meses o algo más estudiando y adentrándose en la transformación digital, me pidió ayuda para crear un prototipo que le permitiera desarrollar su proyecto durante su curso de web development.
Luego de hablar con él y conocer sus inquietudes, me comentó que su intención en un futuro crear una API interbancaria, y hablamos de distintos temas y un poco acerca de la web3, criptos y otros temas relacionados.
Luego de eso busqué crear una interfaz que lo motivara y lo inspirara para mantener su motivación. Mi intención fue crear un wireframe desde 0 e ir explicándole el proceso de desarrollo y cada una de sus etapas.
En el camino ir explicándole cómo estaba estructurada una página web, ademas debatimos sobre distintos stacks tecnológicos y que era lo mas conveniente en su proceso de aprendizaje y desarrollamos un diseño escalable que incluía una landing page, dashboard y sistema de diseño.
Gracias por involucrarme y considerarme en parte de tu proceso de aprendizaje.
Exitos.
Luis Romero
UX | UI - Visual Designer

DESIGN

Figma fue la herramienta de diseño elegida para el proyecto por su capacidad de crear diseños interactivos y prototipos de alta calidad en línea. Además, su funcionalidad de colaboración en tiempo real permitió una comunicación fluida y una fácil adaptación a los cambios necesarios. La eficiencia de Figma en la creación de diseños y su accesibilidad en línea fueron clave para asegurar la calidad y coherencia visual en el proceso de desarrollo.

TECHNOLOGY

HTML y CSS son tecnologías fundamentales en el desarrollo web. HTML es el lenguaje de marcado que permite estructurar y organizar el contenido de una página web, mientras que CSS se encarga de darle estilo y diseño a esa estructura. Ambas tecnologías son esenciales para crear sitios web modernos y atractivos, ya que permiten personalizar la apariencia visual y la funcionalidad de una página. Además, la sintaxis clara y sencilla de HTML y CSS facilita su aprendizaje y su uso en proyectos de cualquier escala.

Se utilizó JavaScript porque permite crear interactividad y dinamismo en la página web, lo que aumenta la experiencia del usuario. Además, es un lenguaje versátil y compatible con la mayoría de los navegadores web, lo que facilita su implementación en el proyecto.

Bootstrap es un framework de diseño web que ofrece una biblioteca de componentes predefinidos y estilos CSS para la creación rápida y eficiente de sitios web. Ofrece una gran cantidad de opciones personalizables, lo que permite a los desarrolladores crear diseños responsivos y atractivos para dispositivos móviles y de escritorio. Bootstrap es ampliamente utilizado en la industria debido a su facilidad de uso, adaptabilidad y rapidez en el desarrollo.

Sass fue utilizado en el proyecto para mejorar la eficiencia y organización del código CSS. Con Sass, pudimos utilizar variables para estilos reutilizables, anidamiento de selectores para escribir CSS de manera más intuitiva y herencia para evitar la repetición de estilos. Esto permitió una mayor facilidad en el mantenimiento y escalabilidad del proyecto. Además, el uso de Sass redujo el tiempo de desarrollo y mejoró la calidad del código final.

Git fue utilizado en el proyecto como herramienta de control de versiones para el manejo eficiente del código fuente y la colaboración en equipo. Con Git, pudimos realizar un seguimiento de las modificaciones realizadas en el código y mantener una historia completa de los cambios realizados, lo que permitió una fácil reversión de cambios en caso de errores. También nos permitió trabajar en diferentes ramas para desarrollar y probar nuevas características sin afectar el código base.

GitHub se utilizó para alojar el repositorio Git del proyecto en línea y permitir la colaboración remota en tiempo real. Con GitHub, pudimos realizar pull requests y revisiones de código, lo que permitió una mejor calidad y colaboración en el proyecto. También se utilizaron las funciones de Issues y Projects para la organización y seguimiento de tareas y problemas del proyecto.

Vercel fue utilizado como plataforma de alojamiento y despliegue para el proyecto. Con Vercel, pudimos automatizar el proceso de compilación y despliegue del código a un entorno de producción en línea. Esto permitió una mayor eficiencia en el proceso de lanzamiento del proyecto y una mayor escalabilidad en caso de un aumento en el tráfico. Además, Vercel ofrece funciones de monitoreo y análisis de rendimiento, lo que permitió una mejor gestión del proyecto y la identificación de posibles problemas.
